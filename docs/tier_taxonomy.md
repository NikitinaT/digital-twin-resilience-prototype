# Tier Taxonomy (Service Activation)

The **tier model** is a *local* prioritization scheme for this prototype (not a claim of compliance with a specific external standard).

## Tier 1 — Critical / Safety-of-Operations
Examples:
- command & coordination / incident response
- security monitoring (SOC)
- data center baseline / core identity & access
- life-safety systems (where applicable)

## Tier 2 — Essential / Mission Support
Examples:
- key teaching buildings (priority classes)
- core admin services for continuity (registrar, payroll)
- limited lab operations (high-value experiments)

## Tier 3 — Limited / Degraded Convenience
Examples:
- dormitories (partial)
- non-critical admin buildings
- discretionary loads

## Output
The policy returns an allocation plan (which tiers get power/connectivity first).
